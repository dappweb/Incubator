# DApp UI 设计方案（V1）

## 1. 产品目标
本 UI 面向三类核心用户：普通会员、节点、超级节点。目标是让用户在 3 步内完成“查看收益 -> 执行操作 -> 验证结果”。

- 强调资金流透明：每笔 USDT 去向可见
- 强调身份权益清晰：普通用户 / 节点 / 超级节点边界明确
- 强调可追溯：奖励来源、分配公式、结算时间可查询

---

## 2. 设计原则

1. 单页单目标：每个页面只承载一个主操作。
2. 先结果后细节：首页先给净收益与风险提示，再看明细。
3. 统一分配表达：所有分红都采用“池子 -> 占比 -> 到账”三段式。
4. 风险前置：所有买入/升级/OTC 卖出操作前弹出规则确认。
5. 状态可验证：实时发放与日结发放必须用不同标签显示。

## 4. 页面级 UI 方案

## 4.1 首页（总览）
**目标**：让用户 5 秒看到“我今天赚了多少、风险状态如何、下一步做什么”。

### 区块 A：账户资产卡（置顶）
- 总资产（USDT估值）
- 今日收益（USDT）
- 累计收益（USDT）
- 可提现（USDT）
- 身份标签：普通 / 节点 / 超级节点

### 区块 B：收益来源占比环图
- 直推奖励
- 节点池奖励
- 超级节点池奖励
- 滑点分红
- 榜单奖励

### 区块 C：规则与风控提醒
- 当前结算模式：实时发放 / 日结发放
- 个人收益上限进度（如 3x 封顶进度条）
- 订单有效状态（生效中 / 已出局 / 已结束）

### 区块 D：快捷操作
- 购买矿机
- 查看我的团队
- 进入 OTC 市场
- 查看昨日奖励榜

---

## 4.2 矿机页
**目标**：完成购买、查看算力、理解分账。

### 区块 A：矿机商品卡
- 单价：100U = 1T
- 可购数量（单次最多10台）
- 预计当日收益（区间）
- 按钮：立即购买

### 区块 B：分账可视化（核心）
采用横向堆叠条 + 百分比标签：
- 60% 进入 ICO/USDT LP 底池
- 5% 直推奖励
- 5% 超级节点池
- 8% 节点池
- 20% 平台池
- 2% 榜单奖励池

### 区块 C：我的矿机订单
表格字段：
- 订单号
- 购买时间
- 数量/T
- 当前状态
- 累计收益
- 收益上限进度
- 操作：查看明细

### 区块 D：奖励到账记录
- 发生时间
- 奖励类型
- 奖励来源订单
- 到账金额
- 到账状态（实时/日结）

---

## 4.3 节点页
**目标**：完成节点身份购买与收益追踪。

### 区块 A：节点身份卡
- 节点价格（USDT）
- 限购：1 个
- 当前库存
- 按钮：购买节点身份

### 区块 B：节点权益卡
- 8% 节点奖池分红说明
- 2% 买入滑点池分红说明
- 2% 卖出滑点池分红说明
- 7% 子币卖出分红说明
- 每项附“计算公式”展开按钮

### 区块 C：团队算力看板
- 我的小区算力
- 全网小区算力
- 我的占比
- 预计下一结算收益

### 区块 D：直推奖励
- 直推层级奖励循环展示（30/20/50）
- 今日直推奖励
- 累计直推奖励

---

## 4.4 超级节点页
**目标**：展示超级节点的高阶权益与团队收益，完成直接购买。

### 区块 A：超级节点身份卡
- 价格（USDT）
- 限购：1 个
- 按钮：购买超级节点

### 区块 B：超级节点权益
- 5% 超级节点奖池（按小区算力加权）
- 8% 节点池叠加权益
- ICO/USDT 手续费池权益
- LIGHT 兑换手续费池（3% / 7%）
- 团队奖与平级烧伤说明

### 区块 C：团队收益排行
- 今日团队新增业绩
- 今日团队收益
- 本周排名

### 区块 D：收益试算器
输入：团队业绩、全网业绩
输出：预估当日可分配 USDT

---

## 4.5 OTC 市场页（节点身份交易）
**目标**：让用户安全、可控地买卖节点/超级节点身份 NFT。

### 区块 A：市场深度
- 当前最高买价
- 当前最低卖价
- 最近成交价
- 24h 成交量

### 区块 B：挂单列表
字段：
- 类型（节点/超级节点）
- 挂单价
- 数量
- 挂单时间
- 状态
- 操作（买入/撤单）

### 区块 C：发布卖单弹窗
- 输入卖价
- 提示：不得低于系统上次成交价
- 提示：卖出扣 10% USDT 手续费进入契约池
- 二次确认：成交后不可撤回

### 区块 D：成交记录
- 成交时间
- 成交价
- 手续费
- 买卖双方地址（中间脱敏）

---

## 4.6 我的页
**目标**：集中管理账户、收益、身份、安全。

### 模块
- 钱包信息（地址、网络、授权状态）
- 身份信息（普通/节点/超级节点）
- 奖励总账（按类型筛选）
- 我的订单（矿机/节点购买/OTC）
- 安全设置（签名授权、风控确认）

## 6. 关键交互流

## 6.1 购买矿机
1. 点击“购买矿机”
2. 输入数量（1-10）
3. 弹出分账预览
4. 钱包签名
5. 成功页显示订单号 + 实时到账记录入口

## 6.2 升级节点/超级节点
1. 进入节点或超级节点页
2. 点击购买
3. 弹窗确认权益与限制（限购1个）
4. 钱包签名
5. 身份标签即时更新

## 6.3 OTC 卖出身份 NFT
1. 选择 NFT 身份
2. 输入卖价
3. 系统校验“不得低于上次成交价”
4. 显示 10% 手续费预估
5. 钱包签名挂单

---

## 7. 状态与异常设计

### 通用状态
- Loading：骨架屏（卡片/表格）
- Empty：引导去首个操作（如“还没有矿机订单”）
- Error：交易失败提示 + 重试按钮
- Pending：链上确认中，显示 tx hash

### 业务异常
- 限购超出（节点/超级节点）
- 挂单价违规（低于上次成交）
- 收益达封顶（不可继续计提）
- 余额不足 / 授权不足

---

## 8. 可视化与文案规范

- 所有金额默认显示：`$1,234.56`
- 百分比显示保留2位：`12.34%`
- 地址显示：`0x12ab...89ef`
- 关键按钮文案统一：
  - 购买矿机
  - 升级节点
  - 升级超级节点
  - 去兑换
  - 查看明细

## 10. 前端实现建议（技术无关）

- 所有分账与收益公式由后端/链上返回结果，前端只做展示，避免口径不一致。
- “实时发放”与“每日发放”字段必须独立，禁止复用一个状态。
- 每个奖励记录需保留 `sourceType`、`sourceId`、`settlementMode`、`txHash` 四个字段用于审计。
